Dynamic Weighting: How AZTH Adapts

How the ensemble monitors multiple signals and adjusts factor exposures based on which signals have been most predictive recently. Compares to ensemble methods in machine learning.

November 2025
8 min read
Adapt

Regime-aware weighting

The Static Problem

Most multi-factor funds combine factors, but they do it statically. They weight value at 30%, momentum at 30%, quality at 40%, and leave it there. The weights don't change. The factors don't adapt. The portfolio assumes all factors are equally predictive all the time.

But factors aren't equally predictive all the time. Value works better in some regimes. Momentum works better in others. Quality shines in downturns, lags in bull markets. Static weighting ignores this reality.

Static multi-factor portfolios are better than single-factor portfolios. But they're not as good as they could be if they adapted to which factors are actually working.

How Dynamic Weighting Works

AZTH doesn't weight factors statically. It monitors how predictive each factor has been recently. It tracks which signals have been working. It adjusts factor exposures based on what's actually happening, not what should happen in theory.

When value has been predictive, value gets more weight. When momentum has been working, momentum gets more weight. When quality signals have been strong, quality gets more weight. The portfolio adapts to regimes, not assumptions.

This isn't market timing. It's signal weighting. You're not betting on which factor will work next. You're adjusting weights based on which factors have been working. It's the difference between guessing and adapting.

The Machine Learning Parallel

Machine learning uses ensemble methods. It combines multiple models. But it doesn't weight them equally. It weights them based on performance. Models that have been accurate get more weight. Models that have been wrong get less. The ensemble adapts.

Dynamic factor weighting works the same way. Factors that have been predictive get more weight. Factors that have been less predictive get less. The portfolio adapts to what's actually happening, not what should happen in theory.

This isn't about predicting regimes. It's about adapting to them. You don't need to know which factor will work next. You need to weight factors based on which have been working.

The Process

AZTH continuously monitors factor performance. It tracks which signals have been predictive over recent periods. It adjusts factor weights based on this data. The portfolio doesn't assume all factors are equal. It adapts to which factors are actually working.

Why This Matters

Static multi-factor portfolios are better than single-factor portfolios. But they leave performance on the table. They weight factors equally even when factors aren't performing equally.

Dynamic weighting captures more of the benefit of multi-factor investing. It adapts to regimes. It weights factors based on what's working, not what should work. It's the difference between a good approach and a better one.

Bottom Line

Multi-factor investing beats single-factor investing. But dynamic multi-factor investing beats static multi-factor investing. AZTH doesn't just combine factors. It adapts their weights based on which factors have been working. It's the difference between combining signals and actually using them.
